# 导入页差异审查说明

## 2026-04 导入页交互与布局修复

- 导入页右侧课程编辑保存/恢复默认时，课程覆盖项的偏好变更不再先触发一次旧预览重建；页面只在新的本地预览完成后刷新一次，避免保存后短暂显示旧右侧详情或重复闪动。
- 导入页右侧课程编辑器的重复逻辑支持 `每 N 天 / 周 / 月 / 年`；只有按周重复时显示多选星期，旧的 `每周 / 双周` 按钮仍作为快捷选择，其中双周等价于每 2 周。按月重复会显示二级选项，可选择“每月在第 N 天”或“每月最后一个星期几”。保存后的规则会继续进入本地预览、重复合并与冲突置顶逻辑。
- 二级重复逻辑编辑器会从同一规则的具体发生项推断每周/双周，即使中间存在节假日或跳周导致间隔是 7 天/14 天的倍数，也不会误显示成单次。重复单位下拉框显示本地化文案而不是 ViewModel 类型名，手动修改间隔、单位、星期或月重复模式会即时刷新摘要和发生项数量。
- 重复日期编辑在开始日期和结束日期之间提供互换按钮；保存时如果开始日期晚于结束日期会自动互换。单次重复逻辑只显示一个日期，从重复规则切回单次时使用开始日期作为单次日期。
- 删除项右侧详情不再直接打开编辑表单；右上操作变为“取消删除”。取消删除会写入一个单次课程覆盖项，把该发生项恢复到本地预览中，后续可按未变更事项继续编辑。
- 顶部“恢复默认”在清除课程编辑与课程展示覆盖前会先关闭当前内联课程编辑器，并在预览刷新后重新使用原始解析值打开详情；恢复默认后的下一次保存不会把上一次编辑结果固化成新的默认值。
- 右侧内联课程编辑器的开始时间、结束时间改为时间输入控件，只接受数字和冒号，并支持 `830` / `0830` 这类省略冒号的输入，失焦后规范化为 `HH:mm`。
- 日期选择下拉中的日期与月份按钮现在使用应用主题色模板，暗色模式下不再继承低对比的系统日历文本颜色。
- 右侧内联课程编辑器的时间输入进一步固定为 `HH:mm` 掩码：冒号是不可删除分隔符，支持只输入小时（如 `6` 保存为 `06:00`）；开始时间按 Enter 跳到结束时间，结束时间按 Enter 结束该时间框编辑。
- 当解析结果或旧覆盖项没有显式日历时区时，导入页课程编辑器和课程设置不再回退到时区列表第一项 `UTC-12`，而是解析为当前程序设置中的 Google Calendar 默认时区；默认配置为 `UTC+8` / `Asia/Shanghai`。
- 时间档未解析但可通过同校区时间档回退确认的项目，会作为单独的时间档确认区置顶显示在左侧变更列表上方，避免被课程分组或普通差异列表淹没。
- 日期下拉日历打开时会通过附加行为把 Popup 内的系统画刷同步为应用主题画刷，后台 UI 回归会读取打开后的日期按钮前景/背景并校验暗色模式对比度不低于 4.5。
- 真正无法解析时间档的未解析课程块也会置顶显示在导入页左侧变更列表上方，和首页“未解析”区保持一致；即使当前筛选为“仅看已选”，未解析确认入口也不会被普通课程变更列表淹没。
- 保存/恢复默认触发的本地预览刷新采用“最新操作优先”：连续保存后立刻恢复默认时，旧的预览重建会被取消，最终只让最新偏好刷新页面；右下角任务框会显示“正在刷新本地预览”的运行任务。
- 导入页内联编辑器的重复逻辑按钮会高亮当前选择或刚手动选择的 `单次 / 每周 / 双周`，避免三个按钮看起来没有状态差异。
- 日期下拉日历会把月份切换箭头、星期标题以及日期按钮文本统一刷新为当前应用主题前景色，深色模式下不再留下灰白混杂或低对比的系统默认文本。
- 导入页按窗口宽度区分 `Compact / Medium / Expanded` 三种布局。
- `Expanded` 宽度显示三段步骤条：`选择变更 -> 预览应用 -> 应用并同步`。
- `Medium` 宽度显示本地化的“变更预览”标题与说明，避免步骤条挤压筛选区。
- `Compact` 与相对较小窗口会更早切换到紧凑布局，并缩小顶部操作按钮、筛选下拉框与搜索框，避免最小窗模式下控件显得过大。
- 顶部摘要卡片中的次操作按钮文案统一为本地化的“同步当前日历”，并直接绑定当前日历预览同步命令。
- “全选当前页”会直接勾选或取消当前筛选结果内的可见变更项，不再只是状态展示。

## 选择与展开行为

- 课程组、规则组、具体发生项统一使用共享的圆角选择样式。
- 课程组与规则组的整行点击层使用透明模板，突出显示由圆角卡片本身绘制，颜色与具体发生项选中态保持一致，不允许默认控件高亮溢出四角。
- 变更课程卡片与规则卡片移除了重复尾部箭头。
- 课程卡片与规则卡片的整个标题区域都可点击展开或折叠，不再只能命中右侧小箭头。
- `i` 信息按钮保持独立点击目标，不会被整卡展开行为吞掉。

## 详情面板规则

- 右侧详情面板现在支持五种选择状态：一级课程组、二级重复逻辑、单个发生项、课程设置、以及置顶未解析项确认。
- 点击一级课程组展开时，右侧显示该课程从当前上传课表解析出的全部重复逻辑；一级详情只列重复逻辑，不展开重复逻辑内的每个具体日期，也不显示“变更摘要”“变更事项”“仅来自解析课表”或“不变详情”等 diff 专用区块。
- 一级课程组右侧详情必须直接从当前上传课程表文件解析出的发生项重新分组生成，不复用左侧 `Changes` 的差异投影，不与 Google Calendar / 本地快照 / provider 预览状态联动，也不显示 `新增 / 更新 / 删除 / 未变更` 等状态标签。解析规则分组必须保留课表周表达式中的间断：例如 `3-9周,11-20周` 不能折叠成一个覆盖缺课周的连续周规则，应拆成两个连续重复段。
- 点击二级重复逻辑展开时，右侧只显示该重复逻辑的具体发生项列表；不显示“变更摘要”、未变更 badge 或“不变详情”卡片。有原始 `Added / Updated / Deleted` 差异的发生项保留选择框并参与应用；未变更发生项作为上下文行显示，可点击查看详情，但不参与应用选择。对重复逻辑执行编辑时应作用于该逻辑覆盖的整组发生项，并在保存后重新生成该逻辑下的具体发生项。
- 当二级重复逻辑已经展开、右侧当前显示的是该逻辑内某个具体日期详情时，再次点击同一个二级重复逻辑标题也必须重新切回该重复逻辑详情，不能因为 `IsExpanded` 状态没有从关闭变为打开而无响应。
- 重复逻辑和单个发生项的修改表单默认不展开；只有点击右侧详情顶部的“修改”后，才在当前详情面板内联展开编辑表单，且表单位于该层级详情列表上方，不打开新页面。
- 点击单个发生项时，右侧显示该具体事项的独立详情；对该事项执行新增、删除、备注、地点、时间等单次编辑时只影响该事项。单次编辑持久化时必须带上原始发生日期，不能只按 `SourceFingerprint` 覆盖整条重复逻辑。若用户在单个发生项编辑器里把重复逻辑改为每周或双周，该保存必须提升为规则级覆盖，清除原始单次绑定，避免保存后又退回“单次”。涉及新增或删除具体事项的保存结果必须重新同步到所属重复逻辑，不能让父级规则摘要继续显示旧范围或旧数量。
- 点击左侧置顶未解析课程块的任意时间/来源行时，右侧必须直接展开同一套内联课程编辑器，并使用解析元数据、已有周次日期和可匹配的时间档作为默认值。保存后该 `SourceFingerprint` 对应的未解析项从置顶未解析区移除，并以新生成的有效发生项进入下方常规变更/已解析课程流。
- 置顶未解析课程块第一次打开时，即使字段保持默认值也必须显示并允许执行“保存”，表示用户确认该未解析项可按当前默认值转为有效发生项；保存后预览重建会把它从未解析区移除，并归入下方常规变更/已解析课程流。
- 当课程或单次发生项编辑把重复逻辑扩展到已有同课程、同日期、同本地时间、同地点的发生项时，预览层必须合并重复发生项，避免同一课程同一时间在下方变更列表中重复出现。若同一日期和时间出现不同课程，则作为冲突日程置顶显示，和未解析项一起计入顶部紫色“冲突”统计卡片。
- 点击一级课程组右侧的 `i` 时不再弹出课程展示编辑器，而是把右侧详情切换到该课程的内联设置详情；即使右侧当时显示的是另一个下拉层级的详情，也必须切换到该 `i` 所属课程的一级课程设置。
- 内联课程设置必须沿用一级课程组右侧详情的结构：先展示时区、Google Calendar 预设颜色等写入选项，再展示该课程从当前解析结果重新分组出的重复逻辑；不要显示“写入选项”等额外卡片标题，不要显示单个发生项详情才需要的“变更摘要”“变更前 / 变更后”或“不变详情”等区块，也不要展开课程、地点、备注等其他编辑字段。
- 内联课程设置目前包含时区、Google Calendar 预设颜色等写入选项。字段改动后右上角显示“保存”；存在已保存覆盖项时显示“恢复默认”。没有未保存改动或已保存覆盖项时，不显示对应按钮。只要存在任何课程编辑、单次编辑或课程展示覆盖的未保存/已保存自行改动，导入页顶部栏都应显示总的“恢复默认”，不应只在当前刚好打开对应修改页时才显示。
- 导入页内联课程编辑卡片的文本输入框应保持紧凑的一行高度；只有标题、地点等文本换行溢出时才增长到多行，避免空输入框占用过多垂直空间。
- 导入差异页的一级课程组列表与右侧详情面板在展开大量重复逻辑或发生项时应使用平滑的像素滚动，不应因为 WPF 逻辑滚动按项目跳动导致鼠标滚轮下滑卡顿。
- 内联课程设置通过工作区偏好持久保存为 per-course presentation override；自动更新 XLS 推导的第一周开始日期时不得丢失这些课程展示覆盖项。
- 右侧详情中的可编辑日程字段应通过当前层级对应的编辑入口进入本地课程编辑器，并持久保存为课程/发生项覆盖；课程展示类字段优先使用内联设置直接保存。
- 新增项与删除项使用单个“详细信息”区块，不再显示空白的左右对比半栏。
- 新增项与删除项的变更摘要只显示状态；除非存在显式冲突或来源提示，不从课程、地点、备注等上下文文本推断字段变更。
- 更新项继续显示结构一致的“变更前 / 变更后”对比。
- `变更前 / 变更后 / 不变详情` 不再把已经包含在 Google Calendar 备注正文中的课程元数据重复拆成独立字段。班级、校区、教师、教学班、课程类型等字段只应保留在备注正文/备注差异里；地点、时间、颜色、时区等真正影响日程行为或 provider payload 的字段仍可单独展示。
- 如果一个更新只改变 Google Calendar 备注正文，右侧下拉层级和变更摘要只显示“备注更新”，不要再把备注里的班级、教师、教学班、课程类型等结构化片段拆成独立字段更新。
- 变更摘要中的字段 badge 只能来自结构化 `ChangedFields`，不能从摘要字符串猜测；例如“未排地点”只作为地点上下文显示，不会在地点字段未变时生成“地点变更”。
- 当某个新增或删除的具体发生项属于当前解析结果里仍然存在、且同一重复逻辑还有其他发生项保留的规则时，二级重复逻辑可按“更新”橙色展示，子项仍保留原始新增/删除身份；后续真正写入 Google Calendar 时仍由 Home 的 provider apply 路径按可 lossless 合并的重复逻辑处理。
- 当一个重复逻辑原本只包含单个发生项，且该发生项被删除时，二级重复逻辑必须显示为“删除”，不能因为父级是规则容器而错误显示为“更新”。

## 备注解析与展示

- 单个发生项详情新增完整 Google Calendar 备注文本差异区，按代码变更样式在同一列中逐行展示删除与新增：删除行使用红色 `-`，新增行使用绿色 `+`，不再分成左右两栏。
- 左右 `变更前 / 变更后` 字段区不再承担主要备注差异展示；备注字段的完整写入文本由专门的差异区负责。
- Google Calendar 备注差异区内的可写备注行直接内联编辑，不再在差异块下方额外显示独立备注编辑框。删除行只读；新增行和不变行中属于用户备注载荷的 `Notes:` 行可编辑；`managedBy:`、`localSyncId:`、`sourceFingerprint:`、`sourceKind:` 等程序管理元数据行始终只读。
- 新增项与删除项不显示备注差异区，因为没有可比较的另一侧备注正文；备注正文只在该项的详细信息中显示。
- 显式 `Notes:` / `备注:` 字段会统一规整成 ` / ` 分隔的展示格式。
- `Notes:` 后面的换行续写内容会继续并入同一条备注展示。
- 斜杠分隔的尾部片段也会并入同一条备注展示。
- 即使源数据没有显式 `Notes:` 标签，只要尾部仍保留人数、考核方式、课程学时组成、学分等信息，导入页也会恢复成可读备注，而不是回退成“无备注”。
- 旧格式备注中的短片段会在显示前补齐标签，例如“考查”规整为“考核方式: 考查”，“理论:32”规整为“课程学时组成: 理论: 32”，课程学时后的裸数字规整为“学分: n”。

## 已解析课程区域

- `Parsed Courses / 已解析课程` 仍作为无差异兜底视图保留。
- 当存在常规 `Added / Updated / Deleted` 差异时，导入页优先保持左侧变更列表、右侧详细信息的主结构。
- 左侧也要显示没有改动的课程，但这些课程应排在变更列表之后，作为已解析课程区域的一部分；当常规差异存在时，该区域只列未变更课程，避免把已变更课程重复展示两次。
- 变更课程头部与兜底解析课程头部的 `i` 信息按钮都切换右侧内联课程设置详情，用于调整时区、Google Calendar 颜色等展示覆盖项。
- 顶部摘要卡片合并了原底部页脚中的可见范围、选择数量和跨页联动状态，底部页脚不再额外占用一块卡片高度。

## 差异分类修复

- 本地快照 diff 不再只按当前 `LocalStableId` 严格分组。它会先按可比较日程键去重，再用同班级、同日期、同来源、同标题、同时间/节次/地点等证据一对一匹配上一版和当前版发生项。
- 课程标题、地点、时间、教师或 source fingerprint 的小幅漂移应优先分类为 `Updated`，而不是拆成 `Deleted + Added`。
- 已有 Google 映射的本地更新会把匹配到的 managed remote event 附回更新行，并同时把 before id 与 after id 视为已计划，避免远端协调再制造一个重复 Added。
- 只有当上一版重复逻辑中确实还有其他当前解析发生项可保留时，删除的子发生项才允许把父级重复逻辑显示为更新；单项规则被整体删除必须保留删除身份。

## 本地化与编码

- 用户可见的导入页标题、筛选选项、状态、回退文本、XAML 标题必须通过 `UiText` 与 `Strings.en-US.xaml` / `Strings.zh-CN.xaml` 提供。
- 筛选、分组和排序逻辑使用稳定的选项索引表示语义，资源字符串只负责显示，避免英文/中文文案变化影响筛选行为。
- 解析源格式所需的中文元数据键可以保留在解析词典或比较辅助逻辑中；这些是源文件标记，不作为 UI 文案直接展示。
- 新增中文 UI 文案时同时更新英文和中文资源字典，避免在 ViewModel、XAML 或测试断言中写入散落的中文字符串，降低编码转换产生乱码的风险。

## 旧版课程展示编辑器遮罩

- 旧版课程展示编辑器覆盖层仍保留兼容，但导入页课程 `i` 的主路径应进入右侧内联设置。
- 背景遮罩使用透明 hover 模板，鼠标移出覆盖层后不会再把整屏刷成蓝色。

## 自动化与回归验证

- 导入页相关自动化依赖稳定的 `AutomationId`，覆盖：
- 导航到导入页。
- 课程组 / 规则组 / 发生项发现。
- “全选当前页”对当前筛选结果的联动。
- `i` 信息按钮切换右侧内联课程设置，课程设置时区/颜色下拉框，以及保存按钮按改动显示。
- 小窗模式下的页面渲染与控件可操作性。
- 回归同时覆盖合成夹具和当前本地配置下的真实导入页场景。
